# Spitex / Bemeda — Megalith UI LLM Review

Public-by-design materials for **soliciting outside-LLM feedback** on the Megalith UI (component consistency, reusability gaps, visual quality, affordance clarity, missing patterns).

This repo is the published copy of `docs/Iterations/01_active/263_*/04_execution/03_llm_feedback/` from the upstream Megalith codebase. Each round is self-contained:

```
round_NNN_<DATE>_<TOPIC>/
  00_brief.md             ← prompt for the LLM (start here)
  README.md               ← round-level checklist + dispatch protocol
  01_context/             ← app + design-system + reusability-audit context
  02_screenshots/         ← 18 PNG captures of representative surfaces
  03_code_samples/        ← actual production code referenced by the brief
  04_responses/           ← per-LLM verbatim responses (filled post-dispatch)
  05_synthesis.md         ← cross-LLM aggregation (filled by the team)
```

## Current rounds

- **`round_001_2026-05-28_component_reuse/`** — first public round, focused on component reuse / standardization gaps (post-iter-271 reusability audit). 18 desktop captures at 1440×3000, signed-in tenant_admin in dev tenant.

## How an LLM should consume this

1. Open `round_NNN/00_brief.md` — that's the prompt. It tells you what questions to answer + what to avoid + what evidence-based responses look like.
2. Read `01_context/*.md` for the framing (app domains, Grok design system principles, reusability audit findings).
3. Examine `02_screenshots/*.png` (numbered 01-18, captioned in `02_screenshots/README.md`).
4. Cross-reference `03_code_samples/` for code-level claims.
5. Write your synthesized response. The team will collect it into `04_responses/<your-provider>.md`.
